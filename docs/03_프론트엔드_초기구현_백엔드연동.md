# 03. 프론트엔드 초기 구현 및 백엔드 연동

## 작업 개요

rest09 바닐라 JS SPA를 뚝딱동화 React 앱으로 포팅하고, 백엔드 API와 연동하는 작업.

## 프론트엔드

### 기술 스택
- React 18 + Vite + React Router v6 + Zustand + Axios
- CSS 변수 기반 디자인 시스템 (rest09 토큰 그대로 이식)
- @phosphor-icons/react (아이콘)
- @supabase/supabase-js (인증)

### 구현된 페이지 (10개)

| 페이지 | 경로 | 상태 |
|--------|------|------|
| MainPage | / | 완료 |
| LoginPage | /login | 임시 (Supabase auth 추후 연결) |
| SignupPage | /signup | 임시 |
| GeneratePage | /create | 완료 |
| StoryCheckPage | /story-check | 완료 |
| LoadingPage | /loading | 완료 (3초 폴링) |
| LibraryPage | /library | 완료 (샘플 동화 5개 기본 표시) |
| ViewerPage | /viewer/:story_id | 완료 |
| MyPage | /mypage | UI만 |
| GuidePage | /guide | 완료 |

### 공통 컴포넌트
- `Button`, `Badge`, `Toast` (context + hook)
- `Illo` (SVG 일러스트 10종: rabbit, ocean, stars, forest, cloud, castle, house, hanok, butterfly, book)
- `Header` (sticky, 테마 토글, 네비 pills)
- `Footer`

### 상태 관리 (Zustand)
- `authStore`: 로그인 유저 상태
- `generateStore`: 동화 생성 입력값 (step, chars, age, setting, situation, moral, artStyle, input_mode)

### axios 설정
- baseURL: `VITE_API_URL` (기본 http://localhost:3000)
- 요청 인터셉터: Supabase 세션 토큰 자동 첨부

### 에셋
- public/assets/에 마스코트·로고 이미지 (rest09에서 가져옴)

## 백엔드

### 추가된 API

#### stories 라우트
| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | /v1/stories | 내 동화 목록 (토큰 없으면 빈 배열) |
| GET | /v1/stories/:story_id | 동화 상세 + pages 포함 |
| DELETE | /v1/stories/:story_id | 동화 삭제 |

#### auth 미들웨어
- `requireAuth`: Authorization 헤더 Bearer 토큰 검증
- Supabase `auth.getUser(token)`으로 유저 확인 후 `req.user` 주입

### pipelineService 개선
- `startGeneratePipeline`이 story 텍스트 생성 후 자동으로 `confirmStory` 호출
- 즉, `/generate` 한 번 호출하면 story → image → completed 전체 파이프라인 자동 실행
- `user_id`를 jobStore에 저장 → stories 테이블 insert 시 반영

### solarService 개선
- child 입력 모드도 연령대별 프롬프트 파일 사용 (`story_child_4_6.txt` 등)
- `fillTemplate()` 추가: 프롬프트 파일의 `{변수}` 플레이스홀더를 실제 입력값으로 치환
- `{moral}` → `{lesson}` 버그 수정

### dalleService 개선
- `generateAllImagePrompts(story, input)`: 연령대별 이미지 프롬프트 파일로 전 페이지 한 번에 생성
- `generateAndSaveImage()`: 내부 GPT 호출 제거, 미리 생성된 prompt 문자열 직접 사용

## 주요 설계 결정

- **로그인 우회**: Supabase auth 미연결 상태에서 로그인 버튼 → 메인 이동 (임시)
- **라이브러리 샘플**: 비로그인/빈 책장 시 rest09 샘플 동화 5개 표시
- **동화 생성 자동화**: 프론트에서 `/generate` 한 번만 호출하면 전체 파이프라인 자동 실행
