### 2. 환경변수 설정
- 루트 `.env` — Supabase URL, Anon Key, API URL
- `server/.env` — Solar API Key, OpenAI API Key, Supabase URL, Service Key, PORT

### 3. 서버 기본 설정
- `server/package.json` — express, @supabase/supabase-js, dotenv, openai
- `server/index.js` — Express 앱 진입점, 라우터 연결
- `server/db/supabase.js` — Supabase 클라이언트 초기화 (service_key 사용)

### 4. 번역 기능 (`feature/translate-api`)
- `server/services/translateService.js`
  - OpenAI GPT-4o-mini 사용
  - 전체 페이지 텍스트를 한 번에 번역 (비용 절감)
  - 이미 번역된 경우 DB 캐시 반환
- `server/routes/translate.js`
  - `POST /v1/stories/:story_id/translate`

### 5. 이미지 생성 + Storage 업로드 (`feature/dalle-api`)
- `server/utils/storage.js`
  - base64 → Supabase Storage `stories-images` 버킷 업로드
  - Public URL 반환
- `server/services/dalleService.js`
  - text_ko → GPT-4o-mini로 영어 장면 묘사 생성
  - gpt-image-1 이미지 생성 (base64 반환)
  - 컷별 병렬 생성 (`Promise.all`)
  - `pages.image_url` DB 저장

### 6. Solar API 스토리 생성 (`feature/solar-api`)
- `server/services/solarService.js`
  - Upstage Solar Pro 사용 (OpenAI 호환 클라이언트)
  - 나이대별 프롬프트 템플릿 로드
  - JSON 형식으로 응답 강제 (`response_format: json_object`)
- `server/prompts/`
  - `story_parent_4_6.txt` — 6컷, 1~2문장/페이지
  - `story_parent_7_9.txt` — 8컷, 3~4문장/페이지
  - `story_parent_10_12.txt` — 10컷, 5~6문장/페이지
  - `story_child.txt` — 6컷, 교훈 없음

### 7. 생성 파이프라인 (`feature/pipeline`)
- `server/utils/jobStore.js` — Map 기반 인메모리 job 상태 관리
- `server/services/pipelineService.js`
  - `startGeneratePipeline` — Solar API 호출 → jobStore 저장 (story_done)
  - `confirmStory` — stories/pages DB 저장 → 이미지 생성 백그라운드 실행
- `server/routes/stories.js`
  - `POST /v1/stories/generate` → job_id 즉시 반환 (202)
  - `GET /v1/stories/jobs/:job_id` → 상태 폴링
  - `POST /v1/stories/:job_id/confirm` → DB 저장 + 이미지 생성 시작
