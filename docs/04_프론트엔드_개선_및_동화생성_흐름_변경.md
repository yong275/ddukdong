# 04. 프론트엔드 개선 및 동화 생성 흐름 변경

## 작업 개요

동화 생성 흐름을 사용자가 줄거리를 확인·수정할 수 있는 방식으로 개선하고,
뷰어·생성 페이지 UI를 디자인 가이드 기반으로 전면 교체.

---

## 흐름 변경

### 이전
```
입력(GeneratePage) → StoryCheck(입력 요약) → Loading(story+image 전체) → Viewer
```

### 이후
```
입력(GeneratePage) → Loading(story 텍스트만)
  → StoryCheck(줄거리 페이지별 확인/수정)
  → Loading(이미지 생성) → Viewer
```

---

## 백엔드

### solarService — plan→write 2단계 파이프라인
- **plan** 프롬프트: 동화 뼈대(장면별 role, scene, emotion, fun_element) 생성
- **write** 프롬프트: plan 기반 실제 본문 작성
- `{story_plan}` 플레이스홀더로 plan JSON 주입
- 캐릭터 외형 설명은 입력값 기반 자동 생성 (`makeCharacterFromInput`)
- 성별 비교 버그 수정: `남자/여자` vs `male/female` 통일

### pipelineService
- `startGeneratePipeline` 에서 자동 `confirmStory` 호출 제거
- story_done 상태에서 대기 → 사용자 확인 후 confirm 호출 시 이미지 생성

### dalleService
- `styleMap` 하드코딩 제거
- `input.art_style_en`, `input.setting_en` 직접 수신하여 DALL-E 프롬프트에 주입

### GET /v1/options
- `server/data/options.js` : 배경/상황/목표/그림체 선택지 데이터
- 프론트에서 앱 시작 시 한 번 fetch, 재배포 없이 선택지 수정 가능

### prompts_v6 반영
- `plan_*.txt` (18개), `write_*.txt` (6개), `image_*.txt` (6개) 업데이트

## 프론트엔드

### StoryCheckPage (전면 교체)
- StoryConfirm.jsx 디자인 기반
- 페이지별 아코디언(`>` 토글) 으로 줄거리 확인
- 각 페이지 텍스트 인라인 수정 가능
- 하단 고정 바: **전체 재생성(최대 3회)** + **확정 후 그림 생성하기**

### ViewerPage (전면 교체)
- storyviewer.jsx 디자인 기반
- 표지 → 내용 페이지(좌:이미지 / 우:텍스트) → 끝 페이지
- 번역 토글 (한국어 ↔ 영어)
- 끝 페이지: 다시보기 / PDF 저장 / 공유하기
- 하단 썸네일 네비게이션, 키보드 ← → 지원

### GeneratePage 개선
- 배경/상황/성장목표: 2단계 선택 UI (1단계 클릭 → 2단계 토글 표시)
- 그림체: `/v1/options` API에서 동적 로드
- 나이대 블록: 컷 수 설명 프롬프트 기준으로 업데이트 (6-8컷, 8-10컷, 12-16컷)
- 인물 카드 3열 고정 가로 배치

### LoadingPage
- `story_done` 상태 감지 시 `/story-check`로 자동 이동

### optionsStore (신규)
- Zustand store, 앱 시작 시 `/v1/options` 한 번 fetch 후 캐싱
